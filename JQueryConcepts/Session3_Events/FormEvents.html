Form Events

    In jQuery, form events are used to handle user interactions with form elements (like input fields, textareas, checkboxes, etc.).
    Here are some commonly used form events in jQuery:

    submit()
    Triggered when a form is submitted.
    Example: Validating form data before submission.

    change()
    Triggered when the value of an input element changes and the element loses focus.
    Example: Updating a preview when a user selects a different option from a dropdown.

    input()
    Triggered every time the value of an input element changes (more immediate than change()).
    Example: Live-updating a character count as the user types in a textarea.

    focus()
    Triggered when an element gains focus (e.g., when a user clicks on an input field or tabs to it).
    Example: Highlighting an input field when it gains focus.

    blur()
    Triggered when an element loses focus (e.g., when a user clicks away from an input field or tabs out of it).
    Example: Validating input when an input field loses focus.

    select()
    Triggered when text within an input or textarea is selected.
    Example: Displaying the selected text length.

    reset()
    Triggered when a form is reset. 
    Example: Confirming with the user before resetting a form.

    invalid()
    Fires when a form element is invalid during validation (required, pattern, type=email etc.).
    Example: Custom handling of invalid form inputs.

    keydown / keyup / keypress (keyboard inside form fields)
    Used to capture user typing inside input/textarea fields.
    Example: Restricting input to numeric characters only.

    cut / copy / paste
    Triggered when user cuts, copies, or pastes into an input/textarea.
    Example: Preventing pasting into a password field.  
    
  Note: These events can be attached to form elements using jQuery's .on() method 
        for better flexibility and to handle dynamically added elements.
