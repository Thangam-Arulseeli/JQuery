jQuery Event handling methods (on(), off()) with Multiple Events + Reset/Re-Enable 
    In jQuery, event handling methods are used to attach and manage event listeners on HTML elements. The two primary methods for event handling in jQuery are .on() and .off().  
on() Method

Used to attach one or more event handlers to selected elements.

Replaces older methods like .click(), .bind(), .delegate().

Supports multiple events, event delegation, and passing data.

off() Method

Used to remove event handlers attached using .on().

Useful when you want to disable certain interactions after a condition.
Supports removing specific handlers or all handlers for an event.
Example Usage
$(selector).on(event, childSelector, data, function)
Attaches an event handler to the selected elements.
- event: The event type (e.g., "click", "mouseover").
- childSelector: (Optional) A selector string to filter the descendants of the selected elements that trigger the event.
- data: (Optional) Data to be passed to the event handler.
- function: The function to execute when the event is triggered.
$(selector).off(event, childSelector, function)
Removes an event handler from the selected elements.
- event: The event type to remove (e.g., "click").
- childSelector: (Optional) A selector string to filter the descendants of the selected elements. 
- function: (Optional) The specific function to remove.

Key Differences Between .on() and .off()
Method	Purpose
.on()	Attach event(s) to elements (supports multiple events and delegation)
.off()	Remove event(s) attached via .on()
