Keyboard Events in jQuery 

    In jQuery, keyboard events are used to handle user interactions with the keyboard (pressing keys, releasing keys, etc.).
    Here are some commonly used keyboard events in jQuery:  
    Keyboard Events in jQuery

      keydown()
      Triggered when a key is pressed down.
      Fires repeatedly if the key is held down.
      Example: Detecting when the Enter key is pressed.

      keyup()
      Triggered when a key is released.
      Example: Performing an action after typing a character.

      keypress()
      Triggered when a key that produces a character is pressed down.
      Doesn’t detect special keys (Shift, Ctrl, Arrow keys, etc.) in modern browsers.
      Example: Detecting when a letter key is pressed. 
      Note: keypress is deprecated, better to use keydown or keyup.

      focus() 
      Triggered when an element gains focus (e.g., when a user clicks on an input field or tabs to it).
      Example: Highlighting an input field when it gains focus.

      blur()
      Triggered when an element loses focus (e.g., when a user clicks away from an input field or tabs out of it).
      Example: Validating input when an input field loses focus.  

      change()
      Triggered when the value of an input element changes and the element loses focus.
      Example: Updating a preview when a user selects a different option from a dropdown.

      input()
      Triggered every time the value of an input element changes (more immediate than change()).
      Example: Live-updating a character count as the user types in a textarea.

  How this works:

        keydown → fires when you press a key (even before the character appears).
        keypress → fires when a character-producing key is pressed (e.g., letters, numbers).
        keyup → fires when you release the key.
        Example Use Cases
        keydown → Detect shortcuts (e.g., Ctrl + S).
        keyup → Form validations after typing.
        keypress → Typing games, live text effects.
        NOTE:
         event.key  -- (modern, recommended  -- Gives the actual character when the key is pressed (a or 1)
         event.which --  (deprecated, old jQuery way) -- Returns the numeric key code of the key pressed. (a -> 97,"Enter" → 13)
        event.keyCode -- (deprecated in modern browsers) -- Returns the numeric code for the key. Similar to event.which, but older DOM standard.
