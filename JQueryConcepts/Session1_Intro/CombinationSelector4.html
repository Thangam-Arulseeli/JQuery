Combination Selectors in jQuery (Group Selector )

    Combination selectors are used when you want to combine multiple simple selectors to be more specific about
       which elements to select.
    They are borrowed from CSS selectors, and jQuery supports them directly.
    
    Here are some commonly used combination selectors in jQuery:

1. Descendant Selector (ancestor descendant)
   Selects all elements that are inside another element (at any depth).
    Example: $("div p") selects all 'p' elements that are inside 'div' elements.
    
2. Child Selector (parent > child)
   Selects all elements that are direct children of a specified element.
    Example: $("ul > li") selects all

3. Multiple Selector (selector1, selector2, ...)
   Selects all elements that match any of the specified selectors.
    Example: $("h1, h2, .highlight") selects all

4. Attribute Selector (element[attribute=value])
   Selects elements that have a specific attribute with a specific value.
    Example: $("input[type='text']") -- selects all  elements with type="text".
              $("src[width='300']")  -- selects all 'src' elements with width="300"

5. Pseudo-class Selector (element:pseudo-class)
   Selects elements based on their state or position. 
    Example: $("a:hover") selects all  elements when they are being hovered over.
    Example: $("li:first-child") selects the first 
    •  element among its siblings.

  6. Grouping Selector (selector1 selector2)
   Selects elements that match a combination of selectors.  
    Example: $("div.highlight > p.intro") selects all 
       elements with class "intro" that are direct children of 
       elements with class "highlight".

      Summary of jQuery Basic Selectors

      In jQuery, selectors are used to "find" (select) HTML elements so we can work with them. The syntax is very simple: $("selector") Here are the three most common basic selectors: $("p") → selects all elements. $("#id") → selects element by id. $(".class") → selects elements by class name. 
      Summary Table
Selector Type	    Syntax	    Meaning
Descendant      	A B	        Selects B inside A (any depth)
Child	            A > B	      Selects B that are direct children of A
Adjacent Sibling	A + B	      Selects the first B immediately after A
General Sibling	  A ~ B	      Selects all B siblings after A

In short:

Use A B → Any depth descendant
Use A > B → Direct child
Use A + B → First immediate sibling
Use A ~ B → All following siblings

 
        What is a pseudo-class?
A pseudo-class is used in CSS to define the state of an element.
It lets you style elements when they are in a particular condition (like when hovered, clicked, visited, etc.).

Syntax:

selector:pseudo-class {
  property: value;
}

Common Pseudo-class Names in CSS
1. User Action States
:hover → When the mouse is over an element.
:active → When an element is being clicked.
:focus → When an input field (or element) is focused.
:visited → Links that have been visited.
:link → Links that have not been visited.

2. Form-related Pseudo-classes
:checked → Radio buttons/checkboxes that are checked.
:disabled → Disabled form elements.
:enabled → Enabled form elements.
:required → Input fields that are required.
:optional → Input fields that are optional.
:valid → Input fields with valid values.
:invalid → Input fields with invalid values.
:in-range → Input values within range.
:out-of-range → Input values outside range.
:read-only → Input fields that are read-only.
:read-write → Input fields that can be edited.

3. Structural Pseudo-classes
:first-child → First child of a parent.
:last-child → Last child of a parent.
:nth-child(n) → The nth child of a parent.
:nth-last-child(n) → The nth child counting from the end.
:only-child → An element that is the only child of its parent.
:first-of-type → First element of its type in a parent.
:last-of-type → Last element of its type.
:nth-of-type(n) → nth element of its type.
:nth-last-of-type(n) → nth element of its type counting from the end.
:only-of-type → Only element of its type.
:empty → Element with no children.
